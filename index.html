Git & GitHub คืออะไร?

📦 Git

โปรแกรมจัดการเวอร์ชัน

  • บันทึกประวัติการแก้โค้ด
  • ย้อนกลับเวอร์ชันได้
  • สำรองโค้ด
  • ใช้งานได้แม้ไม่มีอินเทอร์เน็ต

☁️ GitHub

เว็บไซต์เก็บ Repository

  • Backup Project บน Cloud
  • แชร์โปรเจกต์
  • ทำงานหลายเครื่อง
  • ทำงานร่วมกันหลายคน

🎯 สรุปง่ายๆ

Git = โปรแกรมบนเครื่องของคุณ

GitHub = ที่เก็บบน Cloud

✅ ตรวจสอบการติดตั้ง

git --version

ตัวอย่างผลลัพธ์:

git version 2.55.x

⚙️ ตั้งค่าครั้งแรก

(ทำเพียงครั้งเดียว)

ตั้งค่า User Name และ Email

git config --global user.name "ชื่อ GitHub"
git config --global user.email "อีเมล GitHub"

ตรวจสอบการตั้งค่า

git config --global --list

สร้าง Repository บน GitHub

  • กด New Repository
  • ไม่ต้องติ๊ก Add README, .gitignore, License
  • สร้าง Repository เปล่าๆ

💻 คำสั่งทั้งหมด

🚀 เริ่มโปรเจกต์ใหม่ (ครั้งแรก)

1
เข้าโฟลเดอร์โปรเจกต์
cd C:\Users\Administrator\Documents\AI CITY
2
สร้าง Git Repository
git init
3
เพิ่มไฟล์ทั้งหมด
git add .
4
สร้าง Commit แรก
git commit -m "Initial commit"
5
เชื่อมต่อกับ GitHub
git remote add origin https://github.com/USERNAME/REPOSITORY.git
6
เปลี่ยน Branch เป็น main
git branch -M main
7
Push ขึ้น GitHub ครั้งแรก
git push -u origin main

📋 คำสั่งที่ใช้บ่อยที่สุด

เช็กสถานะ

git status

เพิ่มไฟล์ทั้งหมด

git add .

Commit

git commit -m "ข้อความ"

Push ขึ้น GitHub

git push

Pull จาก GitHub

git pull

ดู Branch

git branch

ดู Remote

git remote -v

ดูประวัติ Commit

git log

ดูประวัติแบบสั้น

git log --oneline

ดูความต่างของไฟล์

git diff

🔄 Clone โปรเจกต์จาก GitHub

git clone https://github.com/USERNAME/REPOSITORY.git
cd REPOSITORY

🔗 เปลี่ยน GitHub Repository

git remote remove origin
git remote add origin https://github.com/USERNAME/REPOSITORY.git
git remote -v

🗑️ ลบ Git ออกจากโปรเจกต์

CMD:

rmdir /s /q .git

PowerShell:

Remove-Item -Recurse -Force .git

เริ่มใหม่:

git init

🔧 แก้ปัญหา

❓ ถ้า Repository มี README อยู่แล้ว

ดึงมาก่อน:

git pull origin main --allow-unrelated-histories

ถ้าไม่ต้องการ README:

git push -u origin main --force

❓ ถ้า Push ไม่ได้

git pull
git push

❓ ถ้า Merge Conflict

1
ดูไฟล์ที่ Conflict และแก้ไขให้เรียบร้อย
2
เพิ่มไฟล์
git add .
3
Commit
git commit -m "Resolve merge conflict"
4
Push
git push

❓ ถ้า Git หาไม่เจอ

เช็กก่อน:

git --version

ถ้าไม่เจอ:

  • Restart VS Code
  • หรือ Restart Windows
